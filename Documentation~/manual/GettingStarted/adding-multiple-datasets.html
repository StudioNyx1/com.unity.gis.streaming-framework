    Adding Multiple Layers

    This section assumes that you have already familiarized yourself with the First Time Setup section and understand how to import the package, create a scene and integrate the UnityGeospatial's core components.

    Creating a New Geospatial Layer

    If you followed the getting started guide, you now have a low-resolution dataset that covers the entire planet. For this tutorial, we will add a high resolution inset.

    1. Download the HillAirforceBase-15cm-v1.0 from the geospatial release folder and decompress the archive anywhere on your machine.
    2. In your asset's right-click context menu select Create -> Geospatial -> Data Sources -> OGC 3DTiles.
    3. Name this newly created asset HillAirforceBase.
    4. Select the asset and, in the inspector, set the URI field of the asset to file:///c:/path-to-dataset/tileset.json
    5. Since this data source is unlit by default, which won't take advantage of the HDRP, let's also force the data source's materials to be lit by changing the Lighting setting from Default to Lit.

    Adding Multiple Layers to the UG System

    We will instruct our UG System to use multiple layers. In the UG System's inspector configure the UG System such that it has two layers: UnityEarth and HillAirforceBase

    Creating a Geodetic Extent

    At this point, if we hit play, we will see both datasets. However, there are areas where the terrain comes out in front of the inset, which doesn't look great. In order to avoid this, we will add a modifier which will cut out the terrain from under our inset. However, before we do so, we need to define the extent that will be used to cut out our terrain.

    1. In the project window's context menu, select Create -> Geospatial -> Data Type -> Geodetic Extent
    2. Give this asset a meaningful name such as HAFB-Extent
    3. Hit play and navigate to the region of interest.
    4. Select our newly created asset and add a new point in the inspector.
    5. Enter Hill Airforce Base's lat/long: 41.109, -111.980
    6. A single point should also appear in the scene view at the entered coordinate.
    7. Form a convex shape by adding points and positioning them in the scene view.
    8. Once you are done editing the extent, make sure to click the Validate & Apply button in the inspector.

    Applying the extent as a modifier

    1. Create an empty GameObject as a child of the UGSystem and call it ExtentModifier
    2. Add an ExtentModifierBehaviour component to the newly created object.
    3. Add this modifier to the UGSystem's modifier stack.
    4. Set the extent to the one we created in the previous step.
    5. Add the terrain layer to the Difference mask.
    6. Add the high resolution inset to the Intersection mask.
    7. Hit play to observe the effect of this modifier.
